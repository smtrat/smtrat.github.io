1 Numbers {#numbers}
2 ====================
3 
4 The higher-level datastructures in CArL are templated with respect to their underlying number type and can therefore be used with any number type that fulfills some common requirements.
5 This is the case, for example, for carl::Term, carl::MultivariatePolynomial, carl::UnivariatePolynomial or carl::Interval objects.
6 
7 Everything related to number types resides in the /carl/numbers/ directory.
8 For each group of supported number types `T`, a folder `adaption_T` exists that contains the following:
9 
10 - Include of the library (if necessary)
11 - Type traits according to @ref typetraits.
12 - Static constants for zero and one.
13 - Operations to fulfill our common interface.
14 
15 From the outside, that is also the rest of the CArL library, only the central numbers/numbers.h shall be included.
16 This file includes all available adaptions and takes care of disabling adaptions if the respective library is unavailable.
17 
18 
19 Adaptions
20 ---------
21 
22 As of now, we provide adaptions of the following types:
23 - CLN (cln::cl_I and cln::cl_RA).
24 - FLOAT_T<mpfr_t>, our own wrapper for mpfr_t
25 - GMPxx, the C++ interface of GMP.
26 - Native datatypes as defined by @cite C++Standard
27 - Z3 rationals.
28 
29 Note that these adaptions may not fully implement all methods described below, but only to some extend that is used.
30 Finishing these adaptions is work in progress.
31

33 Interface
34 ---------
35 
36 The following interface should be implemented for every number type `T`.
37 
38 - @ref typetraits if applicable.
39 - `carl::constant_zero<T>` and `carl::constant_one<T>` if the generic definition from carl/numbers/constants.h does not fit.
40 - Specialization of `std::hash<T>`
41 - Arithmetic operators:
42  - `T operator+(const T&, const T&)` and `T& operator+=(const T&, const T&)`
43  - `T operator-(const T&, const T&)` and `T& operator-=(const T&, const T&)`
44  - `T operator-(const T&)`
45  - `T operator*(const T&, const T&)` and `T& operator*=(const T&, const T&)`
46  - `T& operator=(const T&)`
47 - `bool carl::isZero(const T&)` and `bool carl::isOne(const T&)`
48 - If `carl::is_rational<T>::value`:
49  - `carl::getNum(const T&)` and `carl::getDenom(const T&)`
50  - `T carl::rationalize(double)`
51 - `bool carl::isInteger(const T&)`
52 - `std::size_t carl::bitsize(const T&)`
53 - `double carl::toDouble(const T&)` and `I carl::toInt<I>(const T&)` for some integer types `I`.
54 - `T carl::abs(const T&)`
55 - `T carl::floor(const T&)` and `T carl::ceil(const T&)`
56 - If `carl::is_integer<T>::value`:
57  - `T carl::gcd(const T&, const T&)` and `T carl::lcm(const T&, const T&)`
58  - `T carl::mod(const T&, const T&)`
59 - `T carl::pow(const T&, unsigned)`
60 - `std::pair<T,T> carl::sqrt(const T&)` where the result represents an interval containing the exact result.
61 - `T carl::div(const T&, const T&)` asserting that exact division is possible.
62 - `T carl::quotient(const T&, const T&)` and `T carl::remainder(const T&, const T&)`

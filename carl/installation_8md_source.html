4 Download
5 --------
6 We mirror our master branch to github.com. If you want to use the newest bleeding edge version, you can checkout from https://github.com/smtrat/carl.
7 Although we try to keep the master branch stable, there is a chance that the current revision is broken.
8 You can check [here](https://travis-ci.org/smtrat/carl/builds) if the current revision compiles and all the unit tests work.
9 
10 We regularly tag reasonably stable versions. You can find them at https://github.com/smtrat/carl/releases.
11 
12 Quick installation guide
13 --------------------------------------------
14 - Make sure all @subpage dependencies "dependencies" are available.
15 - Download the latest release or clone the git repository from https://github.com/smtrat/carl.
16 - Prepare the build.
17 @code{.sh}
18 $ mkdir build && cd build && cmake ../
19 @endcode
20 - Build carl (with tests and documentation).
21 @code{.sh}
22 $ make
23 $ make test doc
24 @endcode

30 Using CArL
31 --------------------------------------------
32 CArL registers itself in the CMake system, hence to include CArL in any other CMake project, just use `find_package(carl)`.
33 
34 To use CArL in other projects, link against the shared or static library created in `build/`.
35 
36 Supported platforms
37 --------------------------------------------
38 We have tested carl successfully on the following platforms:
39 
40 - Arch Linux with Clang 3.9.1 and GCC 6.2.1
41 - Ubuntu 14.04 LTS with several compilers on [Travis CI](https://travis-ci.org/smtrat/carl)
42 - OS X 10.11 with several compilers on [Travis CI](https://travis-ci.org/smtrat/carl)
43 
44 We usually support at least all `clang` and `gcc` versions starting from those shipped with the latest Ubuntu LTS or Debian stable releases.
45 As of now, this is `clang-3.8` and newer and `gcc-5` and newer.

1 Logging {#logging}
2 ==================
3 
4 ## Logging frontend
5 
6 The frontend for logging is defined in [logging.h](@ref logging.h).
7 
8 It provides the following macros for logging:
9 - SMTRAT_LOG_TRACE(channel, msg)
10 - SMTRAT_LOG_DEBUG(channel, msg)
11 - SMTRAT_LOG_INFO(channel, msg)
12 - SMTRAT_LOG_WARN(channel, msg)
13 - SMTRAT_LOG_ERROR(channel, msg)
14 - SMTRAT_LOG_FATAL(channel, msg)
15 - SMTRAT_LOG_FUNC(channel, args)
16 - SMTRAT_LOG_FUNC(channel, args, msg)
17 - SMTRAT_LOG_ASSERT(channel, condition, msg)
18 - SMTRAT_LOG_NOTIMPLEMENTED()
19 - SMTRAT_LOG_INEFFICIENT()
20 
21 Where the arguments mean the following:
22 - `channel`: A string describing the context. For example `"smtrat.parser"`.
23 - `msg`: The actual message as an expression that can be sent to a std::stringstream. For example `"foo: " << foo`.
24 - `args`: A description of the function arguments as an expression like `msg`.
25 - `condition`: A boolean expression that can be passed to `assert()`.
26 
27 Typically, logging looks like this:
28 @code{.cpp}
29 bool checkStuff(Object o, bool flag) {
30  SMTRAT_LOG_FUNC("smtrat", o << ", " << flag);
31  bool result = o.property(flag);
32  SMTRAT_LOG_TRACE("smtrat", "Result: " << result);
33  return result;
34 }
35 @endcode
36 
37 Logging is enabled (or disabled) by the `LOGGING` macro in CMake.
38 
39 ## Logging configuration
40 
41 As of now, there is no frontend interface to configure logging.
42 Hence, configuration is performed directly on the backend.
43 
44 ## Logging backends
45 
46 As of now, only the CArL logging backend is supported.
